Executive Summary — ISO 27001 Risk Assessment
This assessment reviews the organisation’s current security risks using the ISO/IEC 27001:2022 framework as the baseline. The purpose is to identify the most important weaknesses, understand how they affect the business, and outline practical steps to reduce overall risk.

The review identified several key gaps across access control, monitoring, and governance. Password practices are weak, MFA is not enforced, and there is no centralised logging or monitoring in place. Patch management is inconsistent, meaning systems may remain vulnerable for long periods. Vendor security reviews are limited, increasing third‑party risk. The organisation also lacks a formal incident response plan, which reduces its ability to react quickly and effectively during a security event.

Overall, the organisation’s risk level is assessed as Moderate to High. The main contributors to this rating are weak access controls, limited visibility into security events, and the absence of structured governance processes.

To strengthen the organisation’s security posture, several improvements are recommended. Enforcing MFA and improving password controls will reduce account‑related risks. Implementing centralised logging and monitoring will improve detection and response capabilities. A structured patching process will ensure vulnerabilities are addressed in a timely manner. Regular vendor assessments will help manage third‑party risks. Finally, developing and testing an incident response plan will ensure the organisation can respond quickly and effectively to future incidents.

These actions will significantly improve the organisation’s overall security maturity and align it more closely with ISO 27001 best practices.
