Recommendations — NIST CSF Gap Analysis
These improvements help the organisation fix the gaps found in the assessment and increase overall security maturity.

1. Asset Inventory
The organisation needs a clear list of all devices, software, user accounts, and cloud services.
This helps identify what needs protection and makes risk assessments accurate.

2. MFA + Strong Passwords
Add multi‑factor authentication to important systems to reduce account takeovers.
Use stronger password rules (longer passwords, no reuse, regular updates) to improve access security.

3. Patching and Updates
Create a simple patching schedule so systems are updated on time.
This reduces the chance of attackers exploiting known vulnerabilities.

4. Centralised Logging and Monitoring
Use one place to collect logs (like a SIEM or log management tool).
This makes it easier to spot unusual activity and respond faster to incidents.

5. Incident Response Plan
Write a basic plan that explains what to do during a security incident.
Include who is responsible, how to communicate, and the steps to contain and recover from an attack.

6. Test Backups Regularly
Backups exist, but they must be tested to make sure they actually work.
Testing ensures the organisation can recover data quickly after an incident.

7. Security Awareness Training
Train staff to recognise phishing emails, suspicious links, and unsafe behaviour.
This reduces human mistakes and strengthens overall security.

Summary
These steps improve the organisation’s maturity across all NIST CSF areas: Identify, Protect, Detect, Respond, and Recover.
They create a stronger, more reliable security posture.
