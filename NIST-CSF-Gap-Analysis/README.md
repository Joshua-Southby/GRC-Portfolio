NIST CSF Gap Analysis

This gap analysis reviews the organisation’s current security maturity using the NIST Cybersecurity Framework. The goal is to identify where the organisation is strong, where gaps exist, and what improvements are needed.

Identify (ID)  
Current maturity: Low
Gaps: No asset inventory, no risk assessment, unclear roles.
Fix: Create an asset list, define responsibilities, perform a basic risk assessment.

Protect (PR)  
Current maturity: Medium
Gaps: Weak passwords, no MFA, inconsistent patching.
Fix: Enforce MFA, improve password policy, set a patching schedule.

Detect (DE)  
Current maturity: Low
Gaps: No centralised logging or monitoring.
Fix: Implement SIEM or log management.

Respond (RS)  
Current maturity: Low
Gaps: No incident response plan or communication process.
Fix: Build and test an IR plan.

Recover (RC)  
Current maturity: Medium
Gaps: Backups exist but aren’t tested.
Fix: Test backup restores regularly.

This analysis shows that the organisation has some basic protections in place but lacks strong detection and response capabilities. Improving MFA, logging, patching, and incident response will significantly increase overall security maturity.
