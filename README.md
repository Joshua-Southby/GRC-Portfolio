This project is a simple, practical example of an ISO/IEC 27001:2022 risk assessment. The aim is to show how an organisation’s security risks can be identified, explained, and prioritised without overcomplicating the process. Everything in this folder is written in a clear, straightforward way so it’s easy to follow, even for someone who isn’t deeply technical.

The folder includes three main files. The Executive Summary gives a quick overview of the organisation’s current risk level and highlights the biggest issues that were found. The Gap Analysis goes into more detail about where the organisation is falling short in areas like access control, monitoring, patching, vendor management, and incident response. The Recommendations file outlines practical steps the organisation can take to reduce risk and improve its alignment with ISO 27001. Together, these files show the full picture: what the problems are, why they matter, and what should be done next.

The purpose of this project is to demonstrate the core skills expected in GRC and ISO 27001 work. That includes spotting weaknesses, understanding how they affect the business, and explaining risks in a way that makes sense to non‑technical people. It also shows the ability to suggest improvements that are realistic and achievable, rather than overly complex or theoretical. This is the kind of work a junior analyst would help produce when supporting an organisation preparing for ISO 27001.

The assessment focuses on the areas that usually cause the most issues in real environments. These include access control, logging and monitoring, patching, vendor risk, incident response, and general governance. These areas were chosen because they come up frequently in audits and interviews, and improving them usually has the biggest impact on reducing risk.

Overall, the organisation’s risk level is considered Moderate to High. The main problems are weak access controls, limited visibility into security events, and missing processes around patching and incident response. The recommendations in this project are designed to help the organisation strengthen its security posture and move closer to ISO 27001 best practices.
