ISO 27001 Risk Assessment — Project Overview
This project is a straightforward example of how an ISO/IEC 27001:2022 risk assessment can look in practice. It’s designed to show the main risks, explain what they mean, and outline the steps an organisation should take to improve its security posture. Everything is written in a clear, simple way so it’s easy to follow without needing deep technical knowledge.

The folder includes several key files that make up the full assessment. The Executive Summary gives a quick overview of the organisation’s current risk level and highlights the most important issues. The Scope explains what parts of the organisation were included in the assessment, while the Methodology describes how the assessment was carried out. The Gap Analysis outlines where the organisation falls short of ISO 27001 expectations, and the Risk Register summarises the main risks in plain language. The Recommendations and Controls files provide practical steps and security measures that help address the issues identified.

The purpose of this project is to demonstrate the core skills expected in GRC and ISO 27001 work. That includes identifying weaknesses, understanding how they affect the business, and explaining risks in a way that makes sense to non‑technical people. It also shows the ability to suggest improvements that are realistic and achievable. This is the type of work a junior analyst would help produce when supporting an organisation preparing for ISO 27001.

The assessment focuses on the areas that usually cause the most issues in real environments, such as access control, logging and monitoring, patching, vendor risk, incident response, and general governance. These areas were chosen because they come up frequently in audits and interviews, and improving them usually has the biggest impact on reducing risk.

Overall, the organisation’s risk level is considered Moderate to High. The main problems are weak access controls, limited visibility into security events, and missing processes around patching and incident response. The recommendations in this project are designed to help the organisation strengthen its security posture and move closer to ISO 27001 best practices.

Conclusion
This assessment gives the organisation a clear understanding of its current security position and highlights the areas that need the most attention. While the issues identified are common, they still create unnecessary risk and make it harder to respond effectively when incidents occur. The recommended improvements — such as stronger authentication, better monitoring, structured patching, vendor reviews, and a formal incident response plan — will make an immediate difference and help build a more reliable security foundation.

By addressing these gaps, the organisation will be in a much stronger position to protect its systems, respond to incidents, and continue working toward full ISO 27001 alignment. The assessment provides a realistic starting point for improving security in a structured and manageable way
